OOPS

Very similar to Java, except that there are no access modifiers like private, protected etc.

Following are similar to Java:

  • Constructor syntax, including the this keyword and its usage.
  • Class definition syntax, which includes attributes and member function definitions.
  • Inheritance and super keyword. 
    class SU extends U {SU() : super();}
  • Some generic classes like List<> are similar to those in Java.
  • abstract class is also used as interface.

extension and mixin

Extensions are a way to add functionality, when needed, to classes already defined and available where it (the extension) is going to be used.

They are defined as follows:

extension A_Ext on String {
    bool isA() => ...
    // Converts a string into a A
    Fraction toA() => A.fromString(this);
}

And used as below:

void main() {
    // str is defined similar to A's string representation
    if (str.isA()) {
        final a = str.toA();
    }
}

The variable being converted (or on which the extension is to be applied) must have the type defined, i.e., this doesn't work with dynamic variables.

Mixins are like classes but those which can provide functionality without inheritance. These do not have a constructor, and any class without a constructot defined can be used as a mixin (but not recommended due to readability issues).

mixin Swimming {
    void swim() => print("Swimming");
    bool likesWater() => true;
}

Defined as above, these can be used with any class to provide additional functionality without inheritance, overriding etc. with keyword is used to use mixins with a class.

class Human with Swimming {
    final String _name;
    final String _surname;
    Human(this._name, this._surname);
    void printName() => "$_name $_surname";
}

Like extension, mixin can also be restricted to a particular class using the on keyword as used in Extensions.
